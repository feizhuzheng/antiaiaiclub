# 复盘日志

记录每一轮：这一版 prompt 长什么样、改写结果在哪一维丢了分、下一版怎么改。

## 基线（原文，未改写）

总分 **78 / 100**。
- 强项：信息密度 23/25（海量硬数据、真名）、语言指纹 12/15（有个人腔调，无 AI 套路词）。
- 弱项：**结构节奏 8/15**——三个板块用同一套"标题问句→数据表→解读"模板复制，排比太齐整，这是最像机器的地方。
- 中游：观点 15、瑕疵真实 11、事实 9。

> 改写的目标不是"换个说法"，而是：**在不损失硬数据和事实可靠性的前提下，把丢分的结构、观点、真实感这几维抬上去。** 风险点是改写很容易把数据改没、把锋利的判断改软——这正是要在 prompt 里防住的。

## 轮次 1：v1 → 59 分（↓19，惨败）

**v1 prompt 思路**：最朴素的"写得像人、口语化、有人情味、句子长短交错"。

**结果**：分数从 78 暴跌到 59。两个致命伤：
1. **信息密度崩了（23→14）**：模型把"口语化"理解成了"把精确数字揉成大概"——383 变"三百多"，所有百分比和真名消失。
2. **观点被磨平（15→9）**："人情味"被执行成了"和气、不得罪人"，所有锋利判断变成"各有各的活法"。
3. 附带灌入大量通用口语填充词（其实/可以说/呢/嘛），反而稀释了原文的个人腔。

**根因**：vague 的"像人"指令，对模型等价于"更亲切、更安全、更圆滑"——而真人度恰恰来自**具体**和**有代价的立场**。prompt 没有保护这两样，模型就顺手把它们当噪音清理掉了。

**v2 怎么改**：
- 加一条铁律：**所有具体数字、人名、专有名词、数据来源一字不改**，改写只动语言、不动事实。
- 明确：**不许软化判断**，原文的锋利立场要原样保留甚至加强。
- 给一个 **AI 高频词/填充词禁用清单**，堵住"其实/可以说/某种程度上/值得注意的是/不仅…而且"这类。
