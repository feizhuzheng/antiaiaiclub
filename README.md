# antiaiaiclub

让 AI 写出来的东西不再像 AI。

这个仓库做一件事：**迭代打磨一个"去 AI 味"的改写 prompt**，用一套 100 分制的"真人度"评分标准来衡量效果，看每一版 prompt 改完之后分数有没有提高。

## 玩法

1. 有一套评分标准 [`RUBRIC.md`](RUBRIC.md)：分数越高，文章越像真人写的（满分 100）。
2. 有一篇测试文章 [`source/original.md`](source/original.md)，先用标准打一个**基线分**。
3. 写一版通用改写 prompt（[`prompts/`](prompts)），它**不包含任何针对这篇测试文章的内容**，对任意文章都能用。
4. 用 prompt 改写文章 → 输出存到 [`outputs/`](outputs) → 用标准打分，存到 [`scores/`](scores)。
5. 复盘（[`RETRO.md`](RETRO.md）：这一版在哪一维丢了分？据此改 prompt，再来一轮。
6. 跑 5 轮，看分数曲线。

> 约束：prompt 必须是通用的。任何"为这篇文章量身定制"的内容都不许进 prompt——否则就是过拟合，换一篇文章就废了。

## 一句话结论

5 轮把"去 AI 味"prompt 从 **59 分（比原文还差）打磨到 88 分（比原文高 10）**。

三条主要心得：
1. **最反直觉**：第一版"努力让它像人/口语化"的 prompt 是唯一让分数下降的，而且降最狠（-19）——因为对模型来说"像人"≈"更软、更圆、更安全"，它会把具体数据和锋利立场当噪音清掉。让 AI 像人的正确方向恰恰相反。
2. **涨分顺序**：先守住具体与立场（别破坏）→ 再打散对称结构（最大的机器指纹）→ 再加有代价的判断和思考痕迹 → 最后做减法磨掉硬塞的人味。
3. **天花板**：单篇改写约在 88 封顶，再高要动文章的数据骨架，超出"改写"范畴。

每轮的 prompt / 改写产物 / 打分 / 复盘分别在 `prompts/`、`outputs/`、`scores/`、`RETRO.md`。所有 prompt 均为通用文体级指令，不含任何针对测试文章的内容。

---
*实验由 Claude (Opus 4.8) 执行；改写与打分均由同一模型完成，打分尽量贴 RUBRIC 客观执行。*
