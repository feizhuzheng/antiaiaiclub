# antiaiaiclub

让 AI 写出来的东西不再像 AI。

这个仓库做一件事：**迭代打磨一个"去 AI 味"的改写 prompt**，用一套 100 分制的"真人度"评分标准来衡量效果，看每一版 prompt 改完之后分数有没有提高。

## 玩法

1. 有一套评分标准 [`RUBRIC.md`](RUBRIC.md)：分数越高，文章越像真人写的（满分 100）。
2. 有一篇测试文章 [`source/original.md`](source/original.md)，先用标准打一个**基线分**。
3. 写一版通用改写 prompt（[`prompts/`](prompts)），它**不包含任何针对这篇测试文章的内容**，对任意文章都能用。
4. 用 prompt 改写文章 → 输出存到 [`outputs/`](outputs) → 用标准打分，存到 [`scores/`](scores)。
5. 复盘（[`RETRO.md`](RETRO.md）：这一版在哪一维丢了分？据此改 prompt，再来一轮。
6. 跑很多轮，看分数曲线（已跑 25 轮：v1–v5 起步，v6–v25 继续逼天花板）。

> 约束：prompt 必须是通用的。任何"为这篇文章量身定制"的内容都不许进 prompt——否则就是过拟合，换一篇文章就废了。

> 📌 **想直接用？** 见根目录 [`PROMPT.md`](PROMPT.md)（= 终版 [`prompts/v25.md`](prompts/v25.md)，完整自包含，复制即用）。`prompts/v1`–`v5` 是完整的；`v6`–`v24` 是迭代记录，只写"本轮在上一版基础上改了什么"的**增量**，不是完整 prompt——完整版只看 v25 / PROMPT.md。

## 结论

25 轮把"去 AI 味"prompt 从 **59 分（比原文还差）一路打磨到 93 分（宽评）/ 88 分（严评）**，并在第 12 轮触顶、之后 13 轮确认平台。

五条主要心得：
1. **最反直觉**：第一版"努力让它像人/口语化"的 prompt 是唯一让分数下降的，而且降最狠（-19）——对模型来说"像人"≈"更软、更圆、更安全"，它会把具体数据和锋利立场当噪音清掉。让 AI 像人的正确方向恰恰相反。
2. **涨分顺序**：先守住具体与立场（别破坏）→ 打散对称结构（最大的机器指纹）→ 加有代价的判断和写实的思考痕迹 → 回捞漏掉的真实数据 → 最后做减法磨掉硬塞的人味。
3. **人味不是越多越好**：第二个对立判断（v8）加分，第三个（v9）凑数减分；一处自然跑题（v3）加分，第二处（v18）冗余；写实的犹豫（v10）加分，编造的生活细节（v23）触护栏翻车。每一类"人味元素"都有边际递减和越界点。
4. **平台期是真的**：v13 之后 13 轮，要么落在评分噪声里、要么触发回撤，没有一次再超过 v12。**单篇纯改写的真人度天花板约 93（宽）/88（严）**，再高得动数据骨架（超出"改写"）或引入第二模型盲评（去掉自评偏差）。
5. **方法论的根本局限**：运动员兼裁判会系统性高估约 5 分（v17）。相对排序可信，绝对分数要打折——要无偏分数必须引入独立的第二个模型盲评。

**通用性**：v20 把终版 prompt 用到一段全新的 AI 腔议论文上，基线 55 → 78（+23），证明它不是过拟合这一篇，奖励的是体裁无关的文体机制。所有 prompt 不含任何针对测试文章的内容。

每轮的 prompt / 改写产物 / 打分 / 复盘分别在 `prompts/`、`outputs/`、`scores/`、`RETRO.md`。
