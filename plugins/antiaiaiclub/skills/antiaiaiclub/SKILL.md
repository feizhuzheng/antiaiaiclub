# antiaiaiclub · 去 AI 味改写

把一篇文章改写得像一个有具体立场的真人写的，去掉 AI 腔，**同时一字不改地保留全部事实、数字、人名、专有名词、数据来源**。

## 第 0 步：拿到原文

- 用户给了**文件路径** → 用 Read 读取全文作为原文。
- 用户**直接粘贴了文字** → 用那段文字作为原文。
- 两者都没有 → 先问用户要原文或文件路径，不要凭空开写。

## 第 1 步：改写

### A. 事实与立场铁律（违反任何一条都算失败）
1. **事实零改动**：所有数字、百分比、人名、公司名、学校名、专有名词、数据来源，一字不改、一个不漏。把精确数据揉成"大概 / 差不多 / 三百多"就是失败。只动语言的组织方式，绝不动事实本身。
2. **不许软化判断**：原文鲜明的、可能得罪人的判断原样保留，能更直接就更直接。禁止"各有各的好""值得辩证看待""为时尚早"这类和稀泥。
3. **禁用 AI 味词句**：其实、可以说、某种程度上、值得注意的是、不仅…而且、综上所述、总而言之、在当今时代、众所周知、不难发现、让我们、（作语气助词的）呢 / 嘛。

### B. 结构形变（AI 最大的破绽是结构太对称）
4. **打破对称**：别让每部分都是"小标题 → 数据 → 解读"一个模子；可合并、打乱、用最反直觉的发现开头，**不要** 01/02/03 编号分节。
5. **段落长短故意拉开**：重点铺长、敢展开；能一句话讲完的就一句话顶一段。
6. **织进一次跑题**：插入一个偏离主线的小观察再自然拉回，要织进正文，别用"（跑题一句…说回正题）"这种生硬脚手架。一篇只用一次。

### C. 语言与节奏
7. **动词与虚字清洗**：软动词（做了 / 进行了 / 对了一遍）换成利落实义动词（核、扒、压、切、接）；删掉"不过话说回来""一种""的一个"这类虚字。腔调要更紧，不是更口语。
8. **短句顿挫**：在最长、信息最密的段落后补一句极短断言（4–12 字），制造长短落差。只用 1–2 处。
9. **腔调统一**：从头到尾是同一个说话的人。

### D. 立场与真实感
10. **两个有代价的判断**：至少给两个明确、排他、作者愿意背书的结论，且二者**维度不同甚至对立**，并点明是不同赛道。都必须建立在数据上——不为对立而对立，也别推到失去依据的偏激（偏激读着像喷子，反伤可信度）。
11. **写实的思考痕迹**：留一次"先迟疑、后笃定"的自我修正（写出当时差点下的错判断），再留一个作者也没想通、数据也答不了的开放问题，并点破这种方法本身的盲区。不要每个论点都完美闭环。
12. **回捞真实数据**：改写常会漏掉原文硬信息，最后过一遍把覆盖率、样本口径这类细节捞回正文。

### E. 不可逾越的红线
13. **C/D 的"人味"只能加在语气、判断、思考过程上。绝不为了显得真实而编造、篡改、模糊任何事实、数据或个人经历。** 编造的瑕疵（虚构的生活细节）比没有更假，会污染全文可信度。

## 第 2 步：自检重改（关键）

用作者的眼睛读一遍，逐条问：
- **密度**：有没有正确的废话？删或换成具体的。原文硬数据漏了吗？捞回来。
- **观点**：最锋利那两句够不够锋利？又被改软了吗？还是推到偏激了？
- **结构**：有没有重新冒出对称、排比、每段等长？
- **语言**：有没有用力过猛的口语、废字？删。腔调是不是同一个人？
- **真实感**：那处瑕疵 / 跑题 / 迟疑是自然长出来的还是硬塞的？硬塞就拿掉，宁缺毋滥。
- **事实**：自检时有没有不小心动了数据？复原。

把最弱的一两维补一刀，过度处减一刀。

## 第 3 步：输出

- 直接输出改写后的**终稿**，不要解释改了什么（除非用户问）。
- 若原文来自文件，输出后问用户是否要把终稿写到新文件（如 `<原名>.humanized.md`），同意再用 Write 写。

> 这套规则是 25 轮迭代打磨出来的；评分标准、迭代过程和局限见仓库 README / RUBRIC.md / RETRO.md。注意：本 skill 是文体级通用工具，对"信息/数据密集型文章"效果最好；纯抒情/虚构类文章酌情使用。
